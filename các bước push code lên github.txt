các bước push code lên github
B1: git init
B2: git add . 
B3: git commit -m"........"
B4: git remote add origin của HTTPS hoặc SSH trong code bên github

các bước đổi tên nhánh thành master -> main 
B1: bit branch -m master main //đổi tên nhánh local
B2: git fetch origin //cập nhật remote
B3: git branch -u origin/main main //set upstream
B4: git push origin main 

*lưu ý: khi hiện lỗi về main -> main (non-fast-forward) thì sử dụng các bước này 
- Muốn giữ code cả 2 bên:
B1: git pull origin main --rebase //lấy code mới về + rebase
B2: git push origin main //push lại 
- Muốn ghi đè code local
B1: git push -f origin main 
B2: git push origin main 



Cấu hình Git lần đầu:
git config --global user.name"quangvudoan"
git config --global user.email"quanggvu24@gmail.com"

Làm việc với repository
git init  // khởi tạo với repo mới
git clone <url> // Clone repo từ Github về máy

Quản lí file & commit
git status //xem trạng thái thay đổi
git add <file> //thêm file vào staging
git add . //thêm tất cả các file
git commit -m"nội dung gợi ý"


Làm việc với nhánh
git branch //liệt kê nhánh
git branch <ten_nhanh> //tạo nhánh mới
git checkout <ten_nhanh> // chuyển sang nhánh
git checkout -b <ten_nhanh> // tạo + chuyển sang nhánh mới
git merge <ten_nhanh> // gộp nhánh vào nhánh hiện tại 

Làm việc với remote (Github)
git remove -v  //xem lại remote hiện đang có
git remove add origin <url> //gắn repo local với Github
git push origin main //push nhánh main lên Github
git pull origin main //kéo code từ github về

Các lệnh lỗi hay thường gặp
git log //xem lịch sử commit
git diff //xem thay đổi chi tiết
git reset --hard <id> //quay lại commit trước
git stash //tạm cất thay đổi chưa commit
git stash pop //lấy lại thay đổi đã stash
